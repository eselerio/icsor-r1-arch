# XGBoost Regressor Summary

## 1. Title and model summary

The xgboost_regressor module implements a measured-space surrogate based on gradient-boosted regression trees using xgboost.XGBRegressor. The repository fits one XGBoost regressor per measured effluent target through a multi-output wrapper, then projects the raw predictions back onto the measured-space invariant subspace defined by the notebook-derived constraint matrix.

## 2. Background and use case

Extreme Gradient Boosting, usually abbreviated XGBoost, is a regularized tree-ensemble method designed for strong predictive accuracy on structured tabular data. It is well suited to the present repository because the ASM1 workflow produces tabular operating variables and measured influent composites, while the surrogate task is nonlinear and may contain interactions that are difficult to capture with a purely linear model.

In this repository the method is used as a fast empirical surrogate for effluent measured outputs. The raw boosting model is statistical, while physical consistency is restored by a separate projection step that enforces the measured-space conservation constraints represented by the A matrix.

## 3. Mathematical definition

For an input vector $x \in \mathbb{R}^{M}$, XGBoost represents one target prediction as an additive tree ensemble

$$
\hat{y}(x) = \sum_{m=1}^{T} f_m(x),
$$

where each $f_m$ is a regression tree and $T$ is the number of boosting rounds. The fitted model minimizes a regularized objective of the form

$$
\mathcal{L} = \sum_{i=1}^{N} \ell(y_i, \hat{y}_i) + \sum_{m=1}^{T} \Omega(f_m),
$$

with squared-error loss in this repository and a tree-complexity penalty $\Omega$. The repository predicts all measured outputs independently with one wrapped XGBRegressor per target, then applies the orthogonal projection

$$
C^{*} = C_{raw} - A^T (A A^T)^{-1} A (C_{raw} - C_{in})
$$

to recover a physically consistent measured-output vector.

## 4. Inputs, outputs, and assumptions

Inputs used by the implementation are HRT, aeration, and the measured influent composites In_COD, In_TSS, In_TN, In_TP, In_NH4_N, In_NO3_N, In_PO4_P, and In_Alkalinity.

Outputs predicted by the implementation are Out_COD, Out_TSS, Out_TN, Out_TP, Out_NH4_N, Out_NO3_N, Out_PO4_P, and Out_Alkalinity.

Assumptions include the following: the training data are generated by the ASM1 workflow, the target space exactly matches the measured-output contract, the XGBoost model is used only as a statistical surrogate, and physical consistency is imposed after prediction by projection rather than by modifying the tree learner itself.

## 5. Implementation used in this repository

Implementation is in src/models/ml/xgboost_regressor.py.

The exact repository workflow is:

1. build the measured-space supervised dataset with shared preprocessing helpers
2. in main.ipynb, create the final train-test split and, when tuning is enabled, a separate Optuna-only subset drawn from the training pool
3. optionally scale the features according to config/params.json
4. optionally run external Optuna on the wrapped XGBRegressor from main.ipynb
5. train the selected model on the notebook-provided training split
6. evaluate both raw and projected predictions with the shared metric and report helpers
7. optionally persist a pickle model bundle and JSON summaries under the configured results paths

The saved bundle stores the fitted estimator, feature and target column order, scaling objects, A matrix, and selected hyperparameters.

## 6. Architecture details and adopted standard architecture name

The adopted architecture is gradient-boosted regression trees, specifically the XGBoost implementation of regularized boosting. Because XGBRegressor is fundamentally a single-target estimator in the usage adopted here, the repository wraps it in sklearn.multioutput.MultiOutputRegressor so one boosted tree ensemble is trained per measured target.

This is not a shared multi-task tree model. The coupling among targets is introduced only after the raw predictions are generated, through the measured-space projection step.

## 7. Training or optimization notes

The Optuna search space now tunes all non-infrastructure XGBoost controls declared for this benchmark: number of trees, learning rate, maximum tree depth, child-weight control, row subsampling, column subsampling, L1 and L2 regularization strengths, and tree-method variant. Only infrastructure constants remain fixed in config/params.json, namely the squared-error objective, random seed, worker count, and verbosity settings, while the global Optuna trial budget remains notebook-owned.

## 8. Prediction workflow

Prediction follows this contract:

1. prepare raw data in measured space or provide prepared feature and constraint-reference frames
2. load the persisted pickle bundle
3. transform the feature columns with the saved scaler when scaling is enabled
4. generate raw multi-target predictions from the wrapped XGBoost estimator
5. project those predictions onto the A-matrix invariant subspace
6. return aligned raw predictions, projected predictions, and constraint references

## 9. Limitations and expected failure modes

Important limitations are that the model remains data-driven, independent target fitting can miss true cross-target structure before projection, and tree ensembles can extrapolate poorly outside the sampled ASM1 operating region.

Expected failure modes include overfitting when the Optuna search budget is large relative to the dataset, unstable behavior if the measured-space basis is changed without retraining, and degraded accuracy if the simulated training distribution is not representative of the intended operating regime.
