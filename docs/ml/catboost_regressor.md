# CatBoost Regressor Summary

## 1. Title and model summary

The catboost_regressor module implements a measured-space surrogate based on catboost.CatBoostRegressor. In this repository the model is configured for multi-target regression with the MultiRMSE loss, and its raw predictions are projected onto the measured-space invariant subspace before the physically consistent outputs are reported.

## 2. Background and use case

CatBoost is a gradient-boosting method based on symmetric, or oblivious, decision trees and ordered boosting ideas that improve robustness on structured data. It is commonly used when strong nonlinear performance is needed with limited manual feature engineering.

Within this repository CatBoost provides a second boosted-tree baseline for the ASM1-to-measured-output surrogate task. It is especially useful as a comparison point because its boosting strategy differs materially from both XGBoost and LightGBM.

## 3. Mathematical definition

The predictor again has additive form

$$
\hat{y}(x) = \sum_{m=1}^{T} f_m(x),
$$

but each $f_m$ is an oblivious tree with the same split rule applied across all nodes at a given depth. The repository optimizes the CatBoost MultiRMSE objective over the full measured-output vector and then applies the same measured-space projection

$$
C^{*} = C_{raw} - A^T (A A^T)^{-1} A (C_{raw} - C_{in})
$$

to enforce the invariant constraints used throughout the project.

## 4. Inputs, outputs, and assumptions

Inputs are the hydraulic and aeration variables together with the measured influent composites derived from the ASM1 composition matrix.

Outputs are the eight measured effluent composites in the repository contract.

The main assumptions are that the training data are generated by the ASM1 simulator, the measured-output basis remains unchanged between training and prediction, and the conservation structure is imposed by post-processing rather than by an internal constrained optimizer.

## 5. Implementation used in this repository

Implementation is in src/models/ml/catboost_regressor.py.

Unlike the wrapped single-target estimators, this repository configuration uses CatBoostRegressor directly with a multi-target loss. All dataset handling, notebook-managed splitting, external tuning, evaluation, reporting, and artifact persistence are delegated to shared helper modules so the model file remains focused on estimator construction and contract-compliant wrappers.

## 6. Architecture details and adopted standard architecture name

The adopted architecture is CatBoost gradient boosting with oblivious trees. The exact repository configuration uses Bayesian bootstrap controls, random-strength regularization, and the MultiRMSE loss function declared in config/params.json.

## 7. Training or optimization notes

Optuna tunes all non-fixed CatBoost training controls declared in this benchmark configuration: number of boosting iterations, learning rate, tree depth, L2 leaf regularization, bagging temperature, and random-strength settings. A small fixed set remains in config/params.json for compatibility and infrastructure reasons (MultiRMSE loss family, bootstrap type, random seed, verbosity, and file-writing disablement). The tuning run is still launched from main.ipynb against a notebook-managed subset.

## 8. Prediction workflow

Prediction loads the saved CatBoost bundle, prepares the measured-space feature frame, applies any configured feature scaling, generates the raw multi-target output, projects that output to the invariant subspace, and returns both raw and projected predictions together with the aligned constraint reference.

## 9. Limitations and expected failure modes

CatBoost still depends on the representativeness of the simulated dataset and can overfit when the tuning space is too large for the available sample count. Prediction quality can also degrade if the model is used outside the operating envelope represented in the ASM1 samples.
